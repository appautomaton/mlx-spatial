## Objective
Start from the parity-first backbone port for MapAnything: use the vendored PyTorch source as the reference, the local `facebook/map-anything` CC-BY-NC 4.0 weights as the quality target, and implement the MLX model core/inference path with numeric parity gates.

## Broader Intent
Get the MapAnything generation/reconstruction pipeline working in `mlx-spatial` using MLX, with the official Desk example and local weights available for smoke testing, while keeping Torch as a reference/dev dependency only.

## Closure Target
Reference system: `vendors/map-anything` plus local `weights/map-anything`.

Closed means the MLX implementation has an explicit MapAnything module boundary, can load or map the relevant local safetensors/config state, matches vendored preprocessing/model-stage behavior through targeted parity checks, and has a first image-only Desk smoke path that demonstrates forward progress toward full inference.

## Key Questions

## Risk Areas

- Full image-to-3D output may require several staged slices; this first change should avoid overclaiming end-to-end parity before core tensors match.
- The best-performance `facebook/map-anything` weights are CC-BY-NC 4.0 and must remain local/reference assets, not bundled package data.

## Constraints
- Keep `mlx-spatial` runtime MLX-first; do not add Torch as a runtime dependency.
- Use `torch-ref` only for reference/parity checks.
- Use `vendors/map-anything` as the source-of-truth implementation reference.
- Use `weights/map-anything` as the local best-performance model-weight target.
- Use `inputs/map-anything/desk` as the official demo-parity smoke input.
- Do not include training, DA3 wrappers, or Apache model support in this change.

## Scope Coverage
- Included: MapAnything MLX module skeleton, config/weight mapping investigation, image preprocessing parity, first model-stage/backbone parity gates, Desk image-only smoke surface, and reference Torch parity checks.
- Deferred: complete all-task MapAnything support, full training pipeline, DA3/external model wrappers, Apache model variant support, commercial/default packaging policy, and distribution of Meta weights.
- Anti-goals: treating Torch as a runtime dependency, bundling CC-BY-NC weights into package artifacts, and producing a standalone gap report instead of closure-oriented implementation.

## Scope Preservation
This intentionally decomposes the full MapAnything MLX inference goal into the first parity-first implementation slice. The broader intent is preserved by choosing boundaries that feed directly into the full image-only inference pipeline.

## Recommended Approach
Approach C: Parity-first backbone port, because MapAnything is large enough that numeric traceability is more valuable than a broad but brittle first pass.

## Key Assumptions and Risks
- The local `weights/map-anything` checkpoint and `config.json` are the correct best-quality model target for this project.
- The official Desk images are sufficient for initial smoke, but not sufficient alone for numeric parity; parity fixtures may need smaller synthetic or captured tensors from the vendored pipeline.
- The first slice may stop before final GLB/3D export if the stable parity boundary lands earlier in the model.

## Rejected or Deferred

- Full end-to-end inference in one slice was deferred because the blast radius is too high for a first MapAnything MLX port.
