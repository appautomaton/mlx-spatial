# Office-Hours Quality

Load this reference only before presenting alternatives, recommending an approach, or writing the approved `INTAKE.md`.

## Anti-Patterns

- Sycophantic validation: praise that does not name evidence.
- Generic alternatives: options that differ by tone but not by tradeoff.
- Premature solutioning: architecture before the problem, stakeholder, or wedge is concrete.
- Category thinking: "users", "teams", or "enterprises" when the workflow needs a named role or observable behavior.
- Soft uncertainty: "could work" without saying what evidence is missing.

## Better Shape

- Name what the answer proves and what it does not prove.
- Ask for observed behavior, real workarounds, or a specific person when the answer stays abstract.
- Make each alternative differ by scope, risk, and learning value.
- End with one concrete next action.

## Prose Hygiene

Office-hours output attracts sycophantic validation and promotional framing. Assessments should name evidence, not feelings.

Scan for:
- "That's an interesting approach": state the evidence-backed assessment
- "There are many ways to think about this": pick one and state what evidence would change it
- "exciting opportunity", "tremendous potential": name the specific wedge and evidence
- Promotional adjectives about the user's idea
- Alternatives that differ by tone but not by tradeoff

Before: "That's a really exciting idea with tremendous potential! There are many ways to think about this interesting problem space."
After: "You named one paying customer (Sarah) and one workaround (the spreadsheet). That's thin demand evidence. Who else would pay, and how do you know?"

## Final Check

If the response sounds encouraging but would not change the user's next decision, revise it.
